# temm-server
B2C 재고 관리 서비스

## 📌 기능적 요구사항

### 1️⃣ 로그인/회원가입
- 유저는 회원가입을 할 수 있다.
- 유저는 회원가입한 정보로 로그인을 할 수 있다.

### 2️⃣ 스토어/장소
- 유저는 스토어를 등록/삭제/수정 할 수 있다.
- 하나의 스토어에는 여러명의 유저가 참여할 수 있다.
- 하나의 스토어에는 여러개의 장소를 가질 수 있다.
- 유저는 장소를 등록/삭제/수정 가능하다
  - 장소는 삭제 시 등록된 물품의 재고가 모두 0이어야한다. 

### 3️⃣ 물품/카테고리
- 유저는 물품을 등록/수정/삭제 할 수 있다. (물품명, 물품 코드 등)
  - 물품 코드는 수정 불가
  - 물품 상태는 "등록", "중단", "삭제"
- 유저는 물품을 검색 할 수 있다. (장소, 카테고리, 이름, 코드 등)

# 📌 비기능적 요구사항

## 1️⃣ 동시성 처리
- 한 물품에 대한 작업(입고, 출고, 조정)은 정합성을 보장하여야 한다. (트랜잭션, Lock 처리)

## 2️⃣ 가용성
- 서버 가용성은 99.9% 이상이어야 한다. (다운타임: 월 43분 이하)

## 3️⃣ 성능
- API 응답 시간:
    - 95% 구간에서 1초 이하
    - 최악의 경우라도 2초 이하
    - 참고: [API Response Time Standards](https://odown.com/blog/api-response-time-standards/)
- 최대 **5,000 명**이 동시에 접속 및 예약할 수 있어야 한다.

## 4️⃣ 유지보수성
- 모든 기능 및 제약사항에 대해 최소 1개 이상의 테스트를 작성해야 한다.

## 5️⃣ 보안
- 모든 API 요청은 토큰 기반의 인증을 거쳐야 한다.
